# A股量化回测系统 - 项目介绍与常见问题

## 项目概述

我开发了一个A股量化回测系统，用 Trae 辅助编码，用 coze 辅助搭建框架。核心是撮合引擎，处理了涨跌停限制、佣金印花税计算、停牌跳过、资金不足判断这些边界条件。策略用双均线交叉，风控模块支持止损止盈和仓位控制。测试用例是我手写的，覆盖了这些边界逻辑。

### 核心功能

- **数据获取**：基于 akshare 获取 A 股真实日线数据（新浪数据源），支持 CSV 本地缓存
- **策略信号**：双均线交叉策略（MA5/MA20），可自定义均线周期
- **撮合引擎**：支持涨跌停限制、佣金印花税计算、停牌处理、滑点模拟
- **风控管理**：支持止损止盈、仓位控制、交易频率限制
- **报告输出**：自动生成资金曲线图和交易明细报告

### 技术栈

- Python 3.11+
- akshare（数据源）
- pandas（数据处理）
- matplotlib（收益曲线可视化）
- pytest（测试）

## 常见问题

### 为什么涨跌停买不了？

**答**：这是 A 股的 10% 涨跌停制度。当股票触及涨停板时，买方力量远大于卖方，通常会封板，导致无法买入；同理，跌停板时卖方力量远大于买方，无法卖出。

在回测中，我们的逻辑是：
- 如果次日开盘价 ≥ 今日收盘价 × 1.1，则不能买入（涨停）
- 如果次日开盘价 ≤ 今日收盘价 × 0.9，则不能卖出（跌停）

### 佣金为什么最低5元？

**答**：这是券商的规定，单笔交易佣金不足 5 元按 5 元收取。

在代码中的实现：
```python
def _calculate_commission(self, amount: float) -> float:
    commission = amount * self.commission_rate
    return max(commission, 5.0)
```

### 夏普比率怎么算的？

**答**：夏普比率 =（策略年化收益 - 无风险利率）/ 策略收益标准差

具体计算步骤：
1. 计算每日收益率序列
2. 计算每日收益率的均值和标准差
3. 年化处理：均值 × 252，标准差 × √252
4. 无风险利率取年化 2%
5. 最终公式：`(mean_return - 0.02 / 252) / std_return × √252`

### 双均线策略为什么亏钱？

**答**：双均线策略在趋势行情中表现较好，但在震荡市中会频繁产生假突破信号，导致多次亏损交易。

**优化方向**：
- 增加成交量过滤：只有在成交量放大时才确认信号
- 添加趋势确认：结合 MACD 或其他指标确认趋势方向
- 调整均线周期参数：根据市场环境优化长短均线周期
- 增加过滤条件：如突破幅度、时间过滤等

### pytest 测了什么？

测试用例覆盖了以下核心逻辑：

| 测试项 | 测试内容 |
|--------|----------|
| 涨跌停限制 | 涨停不能买入、跌停不能卖出 |
| 停牌处理 | 次日成交量为0时跳过交易 |
| 佣金计算 | 单笔佣金不足5元按5元收取 |
| 资金不足 | 资金不足以购买100股时不买入 |
| 撮合逻辑 | 买入卖出的基本撮合流程 |
| 权益曲线 | 资金曲线的计算正确性 |
| 风控模块 | 止损、止盈、仓位限制、交易次数限制 |
| 策略信号 | 双均线交叉信号生成正确性 |
| 数据模块 | CSV保存和加载功能 |

## 使用方式

```bash
# 安装依赖
pip install -r requirements.txt

# 默认参数运行（平安银行）
python main.py

# 指定股票运行
python main.py --stock sh600519 --start 2025-01-01 --end 2026-06-01

# 运行测试
python -m pytest tests/ -v
```
